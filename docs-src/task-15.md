# Task 15: 사용자 피드백 기반 초기 텍스트 컨텐츠 확장 및 수정

## Description
MVP 배포 후 사용자 피드백을 수집하여 초기 텍스트 컨텐츠의 양과 질을 개선하고, 다양한 주제 및 난이도의 텍스트를 추가합니다.

## Details
사용자 테스트 및 피드백을 통해 선호하는 주제, 난이도, 불편사항 등을 파악합니다. 뉴스 기사, 문학 작품, 일상 대화 등 다양한 주제의 한글 텍스트를 수집하고, 이를 `Text` 데이터 모델에 맞게 가공합니다. 텍스트의 길이나 복잡성(어휘 난이도, 문장 구조)을 기준으로 난이도를 분류하여 시스템에 반영합니다.

## Test Strategy
새롭게 추가된 텍스트들이 `TypingTextDisplay`에 올바르게 로드되고, 타이핑 연습 시 통계 계산이 정상적으로 이루어지는지 확인합니다. 다양한 난이도의 텍스트가 의도한 대로 분류되는지 검증합니다.

## Priority
low

## Status
pending

## Subtasks

### Subtask 1: 사용자 피드백 분석 방법론 수립 및 초기 보고서 작성
MVP 배포 후 사용자들로부터 텍스트 컨텐츠 관련 피드백(선호 주제, 난이도 선호도, 불편사항 등)을 효과적으로 수집하고 분석하기 위한 방법론을 수립하고, 이를 바탕으로 초기 분석 보고서를 작성합니다.

**Details:** 피드백 수집 채널(예: 인앱 설문조사, 사용자 인터뷰)을 결정하고, 수집된 데이터를 정량적/정성적으로 분석하는 절차를 명시합니다. 텍스트 주제(예: 뉴스, 문학, 일상 대화), 길이, 난이도에 대한 사용자 선호도를 파악하여 향후 텍스트 수집 및 난이도 분류의 기준을 정합니다. 초기 보고서는 피드백 수집 초기 단계의 결과를 요약합니다.

**Test Strategy:** 최소 10명 이상의 가상 사용자 피드백을 시뮬레이션하여 분석 방법론이 의도한 대로 작동하고 보고서가 정확하게 작성되는지 확인합니다. 피드백 수집 도구가 정상 작동하는지 검증합니다.

**Status:** pending

### Subtask 2: Text 데이터 모델에 난이도 및 주제 필드 추가
기존 텍스트 데이터 모델(예: 백엔드의 `Text` 스키마)에 사용자 피드백을 반영하여 텍스트의 난이도와 주제를 나타내는 필드를 추가합니다. 이는 텍스트 분류 및 필터링 기능의 기초가 됩니다.

**Details:** `Text` 모델(예: `src/models/Text.ts` 또는 데이터베이스 스키마 정의 파일)에 `difficulty` (예: number 또는 enum 타입 '쉬움', '보통', '어려움'), `topic` (예: string 또는 enum 'news', 'literature', 'conversation' 등) 필드를 추가합니다. 필요한 경우 `source` 또는 `author` 필드도 고려합니다. 기존 데이터에 영향을 주지 않는 데이터베이스 마이그레이션 스크립트를 작성합니다.

**Test Strategy:** 데이터베이스 스키마가 성공적으로 업데이트되었는지 확인하고, 새로운 필드에 데이터를 삽입 및 조회할 수 있는지 간단한 CRUD 테스트를 수행합니다.

**Status:** pending

### Subtask 3: 한글 텍스트 복잡성 측정 로직 개발 및 난이도 지표 정의
텍스트의 난이도를 객관적으로 판별할 수 있는 복잡성 측정 로직을 개발하고, 이를 기반으로 '쉬움', '보통', '어려움' 등의 난이도 지표를 정의합니다.

**Details:** 텍스트의 복잡성을 측정하는 기준을 설정합니다. 예를 들어, 문장 길이, 어휘 난이도(희귀 단어 사용 빈도), 문장 구조의 복잡성 등을 고려하여 유틸리티 함수(예: `src/utils/textAnalysis.ts`)를 작성합니다. 이 로직을 사용하여 텍스트에 `difficulty` 값을 할당하는 기준(예: 점수 범위별 난이도 등)을 정의합니다.

**Test Strategy:** 다양한 길이와 어휘 난이도를 가진 샘플 한글 텍스트를 준비하고, 개발된 복잡성 측정 로직이 예상 난이도 점수를 반환하는지 유닛 테스트를 작성하여 검증합니다. 정의된 난이도 지표에 따라 샘플 텍스트가 올바른 등급으로 분류되는지 확인합니다.

**Status:** pending

### Subtask 4: 사용자 피드백 기반 신규 한글 텍스트 컨텐츠 수집 및 가공
사용자 피드백 분석 결과를 바탕으로 뉴스 기사, 문학 작품, 일상 대화 등 다양한 주제와 난이도의 새로운 한글 텍스트 컨텐츠를 수집하고, 저작권 및 사용 가능 여부를 확인한 후 `Text` 데이터 모델에 맞게 가공합니다.

**Details:** 외부 자료(뉴스 웹사이트, 공개 문학 자료, 대화 스크립트 등)에서 텍스트를 수집하고, 저작권 문제를 피하기 위해 공공 도메인 자료를 우선 선택합니다. 수집된 텍스트를 `Text` 모델의 필드에 맞게 정제, 길이 조절, 불필요한 특수문자 제거 등의 전처리 작업을 수행합니다. Subtask 3의 복잡성 측정 로직을 사용하여 초기 난이도를 부여합니다.

**Test Strategy:** 수집된 텍스트 샘플이 `Text` 데이터 모델의 제약 조건을 준수하는지 확인합니다. 가공된 텍스트가 의도한 주제 및 난이도 특성을 잘 반영하는지 수동 검토를 수행합니다.

**Status:** pending

### Subtask 5: 수집된 텍스트 데이터베이스 삽입 및 프론트엔드 텍스트 선택 기능 업데이트
가공된 새로운 한글 텍스트 컨텐츠를 데이터베이스에 대량 삽입하고, 프론트엔드의 `TypingTextDisplay` 컴포넌트에서 이 새로운 텍스트들을 선택하고 불러올 수 있도록 기능들을 업데이트합니다.

**Details:** 데이터베이스 관리 도구 또는 스크립트(예: `scripts/importTexts.ts`)를 사용하여 가공된 텍스트를 데이터베이스에 삽입합니다. 프론트엔드의 Redux store 또는 관련 로직(예: `text` slice)을 수정하여 주제 및 난이도 필터를 통한 텍스트 선택이 가능하도록 API 연동을 업데이트합니다.

**Test Strategy:** 새로운 텍스트들이 백엔드 API를 통해 성공적으로 조회되는지 API 클라이언트로 테스트합니다. 프론트엔드에서 주제 및 난이도 필터를 사용하여 다양한 텍스트를 선택하고, `TypingTextDisplay`에 올바르게 로드되는지 E2E 테스트를 수행합니다. 타이핑 연습 시 통계 계산이 정상적으로 이루어지는지 확인합니다.

**Status:** pending
